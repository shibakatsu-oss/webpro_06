# システム設計書

## 1. システムの対象（概要）
* **システム名**: 一覧表示をベースとしたWebアプリケーション
* **目的**: 一覧表示し，管理しやすくする．
* **主な機能**:
◆REST API（次回以降に説明）による，データの一覧・詳細表示，追加，削除，変更を実装する．データは変数内に記録する．
◆JSファイルの先頭に"use strict";を付けること
◆仕様書はTeXで作成する
◆仕様書は，利用者向け，管理者向け，開発者向けの3部構成とする
◆ソースコードはGithubに置き，レポート内にURLを示すこと
◆授業で説明していない技術については，その概要と採用理由を記載すること

## 2. ページ遷移図
* **トップページ (一覧画面)** (`/`)
    * -> **新規登録ページ** (`/create`) へ移動
    * -> **詳細ページ** (`/detail/:id`) へ移動
* **新規登録ページ** (`/create`)
    * -> 登録ボタンを押すと **トップページ** へ戻る
* **詳細ページ** (`/detail/:id`)
    * -> 削除ボタンを押すと **トップページ** へ戻る
